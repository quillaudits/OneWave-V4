## How to Write an Issue

When submitting an issue, please ensure you include the following information. This structured format helps us quickly understand and address your findings.

### ✅ Format

* **Path & Function:** Specify the file path and the function name where the issue exists.
* **Issue Severity:** Clearly define the **impact** (High/Medium/Low) and **likelihood** (Frequent/Rare).
* **Description:** A clear, concise explanation of the problem.
* **Code Snippet:** Show the specific lines or logic responsible for the vulnerability.
* **Remediation Suggestion:** Propose a fix or a better implementation approach.
* **Proof of Concept (PoC):** If applicable, include a minimal test or interaction flow that reproduces the bug.

## 📮 Where to Submit the Issue

Please submit your issue in the **GitHub Issues** section of this repository. Our team will review and comment directly on your submission.
